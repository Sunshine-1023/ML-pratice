# 房價預測深度學習專案文檔（四版本迭代）

## 1. 專案目標與背景

本專案目標是用結構化房屋資料預測房價中位數 `MedHouseVal`，並在同一資料集上完成從「基線線性回歸」到「MLP 擴容」再到「去除封頂樣本」的完整迭代流程，對比不同建模策略的效果與風險。

專案核心關注點：
- 提升預測準確度（MAE / RMSE / R2）
- 保持訓練流程規範（訓練/驗證/測試分離，避免洩漏）
- 透過可視化（loss 曲線、真值-預測散點）觀察模型行為
- 對 `MedHouseVal=5` 封頂樣本進行專門對比分析

---

- 訓練資料：`数据集/train_data.csv`（16512 筆）

## 3. 通用訓練流程（演算法與建模流程）

### 3.1 前處理
- 使用 `StandardScaler` 對 X、y 標準化。
- 僅使用訓練子集 `fit`，驗證/測試只 `transform`，避免資料洩漏。

### 3.2 指標
- MAE（平均絕對誤差）
- RMSE（均方根誤差）
- R2（決定係數）

### 3.3 早停（適用 V2/V3/V4）
- `max_epochs=5000`
- `patience=80`
- `min_delta=1e-5`
- 監控驗證集 loss，保存 `best_state_dict` 並回載最佳權重。

### 3.4 優化器與損失
- V1 / V2：`MSELoss + Adam(lr=0.01)`
- V3 / V4：`SmoothL1Loss + AdamW(lr=1e-3, weight_decay=1e-4) + ReduceLROnPlateau`

---

## 4. 四個版本的建模迭代

## V1 最初版：線性回歸基線（無早停）
- 模型：`Linear(8 -> 1)`
- 訓練策略：固定 `epochs=2000`
- 切分：直接 train/test（無獨立 val）

## V2 線性回歸 + 8:2 驗證集 + 早停
- 模型：`Linear(8 -> 1)`
- 切分：訓練集再切 `8:2` 為 train/val
- 早停：依 `val_loss(MSE)` 停止

## V3 模型擴大：MLP + 早停（全資料訓練）
- 模型：`MLP(8 -> 64 -> 32 -> 1)`，ReLU + Dropout(0.2)
- 損失與優化：`SmoothL1 + AdamW + LR Scheduler`
- 切分：同 V2（8:2 train/val）+ 獨立 test

## V4 優化方案二：去除封頂樣本後再訓練
- 在 V3 基礎上，先移除訓練集中 `MedHouseVal >= 5` 的樣本
- 過濾後訓練集：15727 筆（移除 785 筆）
- 測試評估同時提供：
  - 全測試集指標（公平對比）
  - 非封頂測試子集指標（觀察主體區間效果）

## 6. 迭代進步解讀（重點）

- **V1 -> V2（僅加早停）**  
  指標不升反降，關鍵原因是實驗設定已改為更嚴格（加入獨立驗證集、早停），且線性模型表達能力有限。

- **V2 -> V3（升級為 MLP）**  
  出現明顯提升：Test R2 由 0.2753 升至 0.7697，RMSE 顯著下降，說明非線性建模能力帶來主要增益。

- **V3 -> V4（去封頂樣本）**  
  驗證集主體區間誤差降低，但在全測試集上 R2 略降（0.7697 -> 0.7594），表示「去封頂」更利於主體區間擬合與視覺分布，但對完整分布泛化略有損失。

結論：
- 若目標是**全測試集整體效果**，V3 最均衡（目前最佳）。
- 若目標是**避免封頂樣本干擾，讓主體區間圖更聚焦**，V4 更合適。

## 8. 後續可改進方向

- 做 K-fold 交叉驗證，降低單次切分波動影響。
- 針對封頂樣本採樣本加權，而非直接刪除，兼顧全分布泛化。
- 引入更強模型（如 LightGBM/XGBoost）作為結構化資料對照組。
- 使用校準/分段誤差分析（低價、中價、高價區間）提升可解釋性。
